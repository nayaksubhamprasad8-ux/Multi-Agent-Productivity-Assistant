SYSTEM ARCHITECTURE

🧠 Primary Agent (Orchestrator)
The primary agent receives all user requests. It uses Claude claude-sonnet-4 to parse intent, plan the workflow, delegate to specialized sub-agents, and synthesize a final coherent response. It maintains conversation history and coordinates multi-step execution.
Model: claude-sonnet-4 · Tools: route_to_agent, synthesize_response, plan_workflow
📋 Task Agent
Manages task CRUD via MCP task-manager server. Reads/writes to tasks table in SQLite.
MCP: task-manager
DB table: tasks
📅 Calendar Agent
Manages events and scheduling via MCP calendar server. Reads/writes to events table.
MCP: calendar
DB table: events
📝 Notes Agent
Manages knowledge capture and retrieval via MCP notes server. Reads/writes to notes table.
MCP: notes
DB table: notes
🗄️ SQLite Database
Structured storage with four tables: tasks, events, notes, agent_logs. Each agent has read/write access to its own table. The primary agent reads from all tables for synthesis. All operations are logged to agent_logs with timestamps and agent identifiers.
⚡ FastAPI REST Layer
The entire system is deployed as an async FastAPI application. Endpoints: POST /chat (main orchestrator), GET /tasks, GET /events, GET /notes, GET /agents/status, POST /workflow. All endpoints return JSON with agent trace metadata.
